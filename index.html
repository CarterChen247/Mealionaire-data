Introduction

Mealinaire is a mobile app developed for explore nearby cuisine based on users' preferences and situation. Due to several invevitable reasons, it has stop operating.

Features & Solution

Use 5 "W"s and 1 "H" to describe the feature and how we implement them:

Who

We are developing a recommender system, recommend proper items to the target user. We focus on people we have interest in exploring delicious foods. How do we know the preferece or taste of an user?

Facebook Android SDK

Facebook Login making the process of user login more easier as well as improving user experiences. We also use the uid from access token to record users' preferences.

Apache Jena

The items(foods) in our database have their own categories, by creating an ontology of the food domain, it can help us to determine the taste of an user in a clear way. After that, we can build the user profile for each user.

What

The system offered cuisine information for user to explore. Those information mainly came from 319papago. The website includes plenty Taiwan restaurant data, and the categories is based on county.

The following is the cuisine data we have currently:

  • restaurants
  • dishes

where

In order to make user more easier to reach their restaurant of interest, we must consider the distance between restaurant and user. The system will show candicate items for user to choose which is ordered by distance from near to far. Introducing a method to calculate the distance between two location on Earth:

SELECT id, ( 6371 * acos( cos( radians(37) ) * cos( radians( lat ) ) * cos( radians( lng ) - radians(-122) ) + sin( radians(37) ) * sin( radians( lat ) ) ) ) AS distance FROM markers HAVING distance < 25 ORDER BY distance LIMIT 0 , 20;

when

In the beginning of building our database, we've already record the running hours of each restaurant in our database. While recommending cuisine to the user, the system will exclude the results which should not be appeared accroding to the user's current time.

why

User who are using our system will receive results when they are using exploring features. How do these results come from?

K-means Algorithm (Clustering)

We use k-means algorithm to train the data, and we'll figure out users who has the similar tastes.

Forecast.io

We use this API to get information such as weather and temperatures, and take these into consideration.

Definition of "Comfortable"

By surveyed the literature which is researching the definitions of "comfortable", the system can recommend appropriate dishes to user considering the weather is rather cold or hot.
